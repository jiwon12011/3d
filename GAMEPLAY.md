# 게임플레이 기획서: 「빗자루 소녀」에 게임을 심는다

기존 DESIGN.md의 "그냥 날아다니는 것 자체가 기분 좋은" 힐링 코어는 그대로 두고,
그 위에 **수집 → 상점 → 해금**의 순한 루프를 얹는다. 목표는 "숙제"가 아니라
"날아다닐 이유가 하나 더 생기는 것".

---

## 1. 디자인 원칙

1. **실패가 없다** — 죽음·체력·시간제한 없음. 코인을 놓쳐도 다시 돌면 그만.
2. **비행이 곧 게임** — 모든 콘텐츠는 "잘 나는 재미"(선회, 저공비행, 구름 뚫기)를 보상한다.
3. **지브리 감성 유지** — 코인은 번쩍이는 금화가 아니라 **별사탕**(작은 별 모양, 은은한 발광). UI는 최소한, 효과음은 딸랑.
4. **5분에 한 번 기쁨** — 하루(주기 5분)마다 뭔가 하나는 얻거나 열리는 페이스.

## 2. 코어 루프

```
날아다닌다 → 별사탕·별사탕 링을 줍는다 → 마을 빵집(상점)에 착륙 →
아이템 구매/빗자루 업그레이드 → 더 멀리·더 빠르게 → 새 지역 해금 → 반복
```

## 3. 수집: 별사탕 (코인)

| 종류 | 배치 | 가치 |
|---|---|---|
| 별사탕 | 길·전깃줄·지붕 위를 따라 낮게 흐르는 줄 (5~8개 체인) | 1 |
| 큰 별사탕 | 시계탑 꼭대기, 대왕나무 가지 사이, 등대 꼭대기 등 "가보고 싶은 곳" | 5 |
| 무지개 링 | 공중 링 코스 (구름 사이, 해안 절벽 따라). 링을 연속 통과하면 콤보 보너스 | 통과당 2, 전체 완주 +10 |

- 획득 연출: 반짝 파티클 + 맑은 종소리(픽업마다 음정이 도레미솔로 올라감 — 콤보의 청각 피드백)
- 리스폰: 하루 주기(5분)가 돌면 재생성. "다 주워서 할 게 없음"을 방지
- 배치는 시드 고정 랜덤 + 수작업 포인트(랜드마크) 혼합. **저공비행·곡예를 유도하는 위치**가 원칙
- HUD: 우상단에 `✦ 128` 하나만. 획득 시 +1 살짝 떠오르는 카운터

## 4. 상점: 마을 빵집 (키키의 구찌오키 빵집 오마주)

- 마을에 **빵집 건물 1채 추가** (차양 + 간판 + 굴뚝 연기, 문 앞에 착륙 매트)
- 빵집 지붕 위 낮은 고도로 진입하면 자동 착륙 → 상점 UI (반투명 크림색 카드, 키보드/클릭 선택)
- 상점 밖에서도 `B` 키로 언제든 보유 목록 확인

### 아이템 목록 (1차)

| 아이템 | 가격 | 효과 |
|---|---|---|
| 빗자루 손질 I·II·III | 30 / 80 / 150 | 기본 속도 12 → 14 → 16 → 18 |
| 부스트 강화 I·II | 50 / 120 | 부스트 30 → 36 → 42, 지속 부담 감소 |
| 리본 염색 (5색) | 각 20 | 빨강→노랑/하늘/보라/분홍/초록 (치장) |
| 원피스 염색 (3색) | 각 40 | 감색→와인/포레스트/네이비 (치장) |
| 반짝이 궤적 | 60 | 비행 궤적에 은은한 별가루 |
| 지지의 방울 | 45 | 고양이 목에 방울 + 선회 시 딸랑 소리 |
| 지역 통행증 | §5 참조 | 새 지역 해금 |

- 치장 아이템은 즉시 착용/해제 가능 — "모으는 재미"의 중심
- 가격은 "10분 비행 ≈ 별사탕 60~80개" 기준으로 책정 (첫 업그레이드는 5분 내 도달)

## 5. 지역 해금

현재 월드를 4개 구역으로 나누고, 시작은 마을 주변만 열려 있다.

| 지역 | 내용 | 해금 조건 |
|---|---|---|
| ① 언덕 마을 (시작) | 마을, 시계탑, 빵집, 논밭 | — |
| ② 깊은 숲 | 대왕나무, 호수, 반딧불이 군락 | 통행증 100 ✦ |
| ③ 바닷가 | 등대, 해안 절벽 링 코스 | 통행증 250 ✦ 또는 배달 5회 |
| ④ 하늘 위 | 구름 위 무지개 링 대코스, 별사탕 밭 | ②③ 모두 해금 후 500 ✦ |

- **잠긴 지역의 표현**: 벽이 아니라 **짙은 뭉게구름 커튼**. 접근하면 부드럽게 밀려나며
  "아직 갈 수 없어요 — ✦100 통행증" 말풍선. 세계관을 깨지 않는 소프트 락
- 해금 연출: 구름 커튼이 갈라지며 빛이 쏟아지고, 새 지역으로 카메라 팬 1회

## 6. 배달 퀘스트 (선택 콘텐츠, 2차)

키키답게 — 빵집에서 소포를 받아 **불 켜진 창문의 집**에 배달:

- 빵집 착륙 → "소포 배달해줄래?" → 목표 집에 은은한 빛기둥 표시
- 목표 집 지붕 근처로 저공 접근하면 배달 완료, 보수 ✦15~30 (거리 비례)
- 시간제한 없음, 거절 가능. 밤에는 반딧불이가 길을 안내
- 배달 5회마다 빵집 아주머니의 감사 선물 (한정 치장 아이템)

## 7. 저장

- `localStorage`에 JSON 1키: `{ candies, upgrades, cosmetics, regions, deliveries }`
- 픽업/구매/해금 시마다 즉시 저장. 코드 버전 필드로 마이그레이션 대비

## 8. 파일 구조 (추가분)

```
src/
  game/
    save.js        — localStorage 로드/저장/마이그레이션
    candies.js     — 별사탕·링 배치/리스폰/픽업 판정(InstancedMesh + 거리체크)
    regions.js     — 구역 정의, 구름 커튼, 소프트 락 밀어내기, 해금 연출
    shop.js        — 빵집 착륙 판정 + 상점 UI(DOM 오버레이) + 아이템 효과 적용
    delivery.js    — 배달 퀘스트 상태머신 (2차)
    hud.js         — 별사탕 카운터, 말풍선, 토스트
```

- 픽업 판정은 매 프레임 전수 검사 대신 **공간 그리드(셀 20m)** 로 주변 셀만 검사
- 별사탕 전체는 InstancedMesh 1개 (회전·발광은 셰이더 time 기반) — 드로우콜 +1

## 9. 구현 순서

1. **1차 (코어 루프)**: save → candies(별사탕+HUD) → shop(빵집 건물+업그레이드 2종+치장 2종) → regions(② 숲 해금 1개)
   → 여기까지만 해도 "게임"이 된다. 플레이 테스트로 가격/페이스 튜닝
2. **2차**: 무지개 링 코스, ③④ 지역, 나머지 아이템, 해금 연출
3. **3차**: 배달 퀘스트, 한정 선물, 사운드 폴리시

## 10. 리스크 / 지킬 것

- 상점 UI가 게임의 톤을 깨지 않게 — DOM 오버레이도 시작 화면과 같은 크림색 종이 질감
- 별사탕을 너무 촘촘히 깔면 "청소"가 된다 — 줄 사이 여백을 넉넉히, 콤보는 링 코스에만
- 업그레이드 최고 속도(18)에서도 조작감(damped lerp)이 무너지지 않는지 확인
- 모바일: 상점 선택은 터치 대응 필수
