# 기획서: 지브리풍 3D 빗자루 탐험 세계 「빗자루 소녀」 (jiwon12011/3d)

## 1. 컨셉

**마녀 배달부 키키**처럼 빗자루를 탄 소녀의 3인칭 시점으로, 토토로·포뇨 같은 지브리 감성의 세계(작은 마을, 언덕, 숲, 바다)를 자유롭게 날아다니는 브라우저 탐험 게임.
목표·미션 없음 — "그냥 날아다니는 것 자체가 기분 좋은" 힐링 경험이 전부.

- 레포: `jiwon12011/3d` (현재 README.md만 있는 빈 레포 → 처음부터 제작)
- 기술: **Three.js** (CDN import map, 빌드 도구 없음, 바닐라 JS 모듈)
- 외부 3D 모델/텍스처 **전혀 없이** 프리미티브 조합 + 코드 생성 → 로딩 0초, GitHub Pages로 바로 배포 가능

## 2. 지브리 느낌의 핵심 분석 (참고 이미지 4장 기준)

지브리 배경미술이 "지브리처럼" 보이는 이유를 분해하면, 구현해야 할 것이 명확해진다:

| 요소 | 지브리의 특징 | 3D로 옮기는 방법 |
|---|---|---|
| **구름** | 화면의 주인공. 크고 낮게 깔린 뭉게구름, 밑면은 푸른 회색 음영 | 구체 클러스터 + 툰 셰이딩 (§4) |
| **색** | 채도 높은 초록·하늘색, 그림자조차 탁하지 않고 청보라빛 | 툰 그라디언트맵의 어두운 단계를 "검정"이 아닌 "차가운 청록"으로 (§3.2) |
| **빛** | 따뜻한 노란 햇살 + 하늘에서 내려오는 푸른 환경광의 대비 | 웜 DirectionalLight + 쿨 HemisphereLight (§3.4) |
| **음영** | 2~3단계로 뚝뚝 끊기는 셀 음영, 부드러운 경계 | MeshToonMaterial + 3단계 gradientMap (§3.1) |
| **원경** | 멀수록 채도가 빠지고 하늘색에 녹아듦 (공기원근법) | 수평선 색과 일치시킨 Fog (§3.3) |
| **형태** | 디테일보다 덩어리감. 나무는 브로콜리 같은 뭉게 실루엣 | 로우폴리 구체 뭉치, 디테일은 색으로만 (§7) |
| **생활감** | 전봇대, 전깃줄, 빨래, 굴뚝 연기 같은 소품 | 전깃줄 커브, 연기 파티클 (§6) |
| **움직임** | 항상 무언가 흔들림 — 잎, 풀, 물결 | 버텍스 셰이더 바람 + 나뭇잎 파티클 (§8) |

## 3. 렌더링 파이프라인 (지브리 룩의 기술적 핵심)

### 3.1 툰 셰이딩
- 모든 재질은 `MeshToonMaterial` + 공용 **3단계 gradientMap**
- gradientMap은 4×1 `DataTexture` (`NearestFilter` 필수 — 안 하면 보간돼서 툰 느낌이 죽음)
- 단계 값 예: `[0.55, 0.75, 1.0, 1.0]` — 어두운 면도 55% 밝기 이상 유지해서 "그림자도 밝은" 지브리 특유의 화사함을 만든다. 어두운 단계가 0.3 이하로 내려가면 즉시 '일반 게임' 느낌이 되므로 주의.

### 3.2 색 팔레트 (`palette.js`에 상수로 집중 관리 — 튜닝의 중심)
```
하늘 천정   #3d7edb   하늘 중간   #7ec3ea   수평선     #dff3f5
구름 밝은면 #ffffff   구름 밑면   #b8c8dd
잔디 밝음   #8fd05a   잔디 기본   #6ab53f   잔디 그늘  #4a9a3a
나뭇잎     #4e9c3e / #66b34c / #3f8a46 (3종 변주)
나무 줄기   #8a6349
지붕       #c0574f / #d97f4e / #7f9bb3 (테라코타/주황/청회색 변주)
벽         #f2e8d5 / #f7f0e0 / #e8d9c0 (크림 변주)
바다/호수   #4aa8c9 (얕은 곳 #7fd4dd)
길         #d9c9a3 (흙길 크림빛)
캐릭터     원피스 #2e2a3d(감색), 리본 #d43d3d, 머리 #5a3825, 피부 #ffe0c2
```
- 렌더러: `outputColorSpace = SRGBColorSpace`, **toneMapping은 NoToneMapping** (ACES는 채도를 빨아먹어서 지브리색이 죽는다. 대신 팔레트 자체를 완성색으로 잡는다)

### 3.3 안개 (공기원근법)
- `scene.fog = new THREE.Fog(수평선색 #dff3f5, near 120, far 600)`
- 하늘 셰이더의 수평선 색과 **정확히 같은 색**이어야 원경 언덕이 하늘에 자연스럽게 녹는다 — 이게 어긋나면 원경에 띠가 생기며 가짜 티가 남

### 3.4 조명
- `DirectionalLight`: 색 `#fff4d6`(따뜻한 햇살), intensity 1.6, 위치는 비스듬한 오후 태양 각도. 그림자 캐스터는 이것 하나만
- `HemisphereLight`: sky `#bfd9ff` / ground `#9bc47a`, intensity 0.9 — 그늘면에 하늘의 푸른기가 스며들게 (지브리 그림자의 청색기)
- 그림자: `PCFSoftShadowMap`, 2048 맵 1장, 플레이어 주변만 따라다니는 shadow camera (넓은 월드 전체를 커버하지 않음)
- 아웃라인은 **넣지 않는다** — 지브리 배경미술은 선이 거의 없는 수채 덩어리감이 본질

## 4. 하늘과 구름 (이 게임의 주인공)

### 하늘
- 카메라를 따라다니는 대형 반구 + 커스텀 `ShaderMaterial`
- 3색 세로 그라디언트(천정→중간→수평선), `pow()` 커브로 수평선 근처를 넓고 밝게 — 참고 이미지 3번(도시 계단)의 하늘 배분
- 태양 방향에 은은한 밝은 번짐 한 점

### 뭉게구름 (통과 가능!)
- 구름 1덩이 = 6~14개의 구체(스케일 랜덤, 아래는 평평하게 y스케일 0.6)를 뭉친 Group
- 재질은 흰 MeshToonMaterial — 툰 3단계가 자동으로 "윗면 순백 / 밑면 푸른 회색" 음영을 만들어줌. 이게 지브리 구름의 핵심
- 고도 60~120에 30~40덩이 배치, 각자 아주 느리게 수평 드리프트 (구름마다 속도 다르게)
- 플레이어가 구름을 **통과해서 날 수 있음**: 통과 시 화면이 잠깐 뿌예지는 연출(안개 near를 순간 좁혔다 복귀) — 키키의 구름 뚫는 장면
- 지면에 구름 그림자는 생략 (성능/복잡도 대비 효과 낮음)

## 5. 지형과 물

- `PlaneGeometry` 800×800 (128×128 세그먼트) + 심플렉스 노이즈 2옥타브로 완만한 언덕. 마을 지역은 노이즈를 눌러 평평하게 (flatten mask)
- **버텍스 컬러**로 잔디 톤 변주: 높이·노이즈에 따라 밝은 초록↔진초록 얼룩 — 단색 초록 평면은 즉시 싸구려 느낌이 나므로 필수
- 월드 한쪽 가장자리는 바다(포뇨): 별도 평면, 버텍스 셰이더로 느린 물결 + 얕은 곳은 밝은 청록 그라디언트, 반짝이는 흰 점 스프라이트 몇 개
- 언덕 사이에 작은 호수 1개, 흙길이 마을↔논밭↔큰나무를 잇는다 (지형 위에 밝은 크림색 리본 메시)

## 6. 마을 (키키의 코리코 + 토토로 시골의 혼합)

- **언덕 위 마을** (~18채): 유럽풍 — 크림색 벽 Box + 프리즘 지붕(색 3종 변주), 굴뚝, 창문은 짙은 색 면 + 몇 집은 은은한 노란 emissive(불 켜진 창). 크기·회전·색을 랜덤 변주해서 "같은 집 복붙" 느낌 제거
- **랜드마크**: 마을 중앙 시계탑 1개(멀리서 방향 잡는 기준점), 바닷가 절벽에 등대 1개(포뇨)
- **시골 구역**: 논밭 패치(연두 격자 평면), 원두막, 버스정류장 표지판(토토로)
- **전봇대 + 전깃줄**: 길을 따라 6~8개, 전깃줄은 `QuadraticBezierCurve3`로 자연스럽게 처지는 커브 → 얇은 TubeGeometry. 참고 이미지 4번의 생활감을 만드는 일등공신
- 굴뚝 2~3개에서 느리게 피어오르는 연기 (구체 몇 개가 올라가며 커지고 투명해지는 파티클)

---

## 7. 자연

- **나무**: 줄기(실린더) + 잎뭉치(찌그러뜨린 Icosahedron 2~4개, 잎색 3종 변주) = 브로콜리 실루엣. `InstancedMesh`로 300~500그루를 드로우콜 2개에 처리
- **대왕 녹나무 1그루** (토토로의 나무): 일반 나무의 8배 크기, 숲 한가운데 랜드마크. 주변을 선회 비행하는 재미 포인트
- **풀·꽃**: 교차 쿼드 InstancedMesh 수천 개, 마을 주변·길가에 밀도 높게
- **바람**: 나무 잎뭉치와 풀에 `onBeforeCompile`로 버텍스 셰이더 주입 — `sin(time + worldPos)` 기반으로 위쪽 버텍스만 살랑살랑. 지브리는 "멈춰있는 자연"이 없다

---

## 8. 파티클 연출

- **흩날리는 나뭇잎**: 화면 주변에 항상 10~20장, 나선을 그리며 낙하 (참고 이미지 4번) — 초록 쿼드 인스턴스
- **부스트 바람 줄기**: Shift 부스트 시 카메라 주변에 흰 스피드라인 (얇고 긴 반투명 평면들이 뒤로 흐름)
- **구름 통과 뿌옇게**: §4 참조

## 9. 플레이어 캐릭터 (프리미티브 조립)

빗자루 탄 소녀 + 검은 고양이. 모든 파츠는 프리미티브:
- 머리(구) + 갈색 단발머리(반구 덮개) + **빨간 리본**(납작 Box 2개) — 키키의 상징
- 감색 원피스(원뿔을 눌러 치마처럼) + 팔(캡슐, 빗자루를 쥔 각도) + 다리·빨간 신발(옆으로 모아 앉은 자세)
- 빗자루: 손잡이(긴 실린더, 나무색) + 솔(원뿔 + 짚색, 끝 살짝 벌어짐) + 손잡이에 매단 작은 가방(배달부!)
- **지지(검은 고양이)**: 빗자루 뒤쪽 솔 위에 앉음 — 검은 구 2개 + 귀 원뿔 + 흰 눈, 꼬리(가는 실린더)가 좌우로 살랑임

**애니메이션** (본 없이 코드로):
- 부유 bobbing: 전체가 sine으로 상하 ±0.15
- 선회 시 뱅킹: 기체 roll 최대 35°를 부드럽게 lerp
- 부스트 시 몸을 앞으로 숙임(pitch -10°), 치마·머리끝 살짝 펄럭(스케일 진동)

## 10. 비행 시스템

| 항목 | 값/방식 |
|---|---|
| 기본 속도 | 12 m/s 자동 활공 (빗자루는 멈추지 않는다 — 조작 부담 제거) |
| 부스트 | Shift 홀드 시 30 m/s, 가감속은 exp smoothing |
| 조작 | `W/S·↑/↓` 피치(상승·하강 ±45° 제한), `A/D·←/→` 선회(yaw) |
| 회전 | 목표 각도로 damped lerp — 즉각 회전 금지, 빗자루 특유의 스윽~ 미끄러지는 관성 |
| 고도 | 지형 높이 + 1.5m 아래로 못 내려감 (노이즈 함수 재사용으로 높이 샘플링), 상한은 구름 위 150m |
| 경계 | 월드 끝 접근 시 부드럽게 안쪽으로 선회 유도 |

**카메라**: 빗자루 뒤 6m·위 2.5m를 exp damping으로 추적. 선회 시 카메라도 살짝 같이 롤(뱅킹의 절반). 부스트 시 FOV 60→72 확장으로 속도감. 시작 시 마을 전경 → 캐릭터 뒤로 스윽 들어오는 인트로 카메라 무브 1회.

**UI**: 시작 오버레이(제목 「빗자루 소녀」, 조작법, "클릭해서 날기 시작") — 부드러운 크림 배경, 클릭 시 페이드아웃. 인게임 UI는 없음(몰입 우선).

## 11. 파일 구조

```
3d/
  index.html          — import map(three CDN), 캔버스, 시작 오버레이
  src/
    main.js           — 부트스트랩, 리사이즈, 메인 루프
    palette.js        — §3.2 색상표 + 공용 gradientMap + 툰 재질 팩토리
    noise.js          — 심플렉스 노이즈 (지형·배치·바람 공용)
    world/
      sky.js          — 하늘 셰이더 반구 + 태양
      clouds.js       — 뭉게구름 생성/드리프트/통과 연출
      terrain.js      — 언덕 지형 + 버텍스컬러 + 높이 샘플러 + 바다/호수/길
      town.js         — 집/시계탑/등대/전봇대/전깃줄/연기
      nature.js       — 나무(인스턴싱)/대왕나무/풀꽃/바람 셰이더
      particles.js    — 나뭇잎/스피드라인
    player/
      broom.js        — 캐릭터 조립 + 파츠 애니메이션
      controls.js     — 입력/비행 물리
    camera.js         — 추적 카메라 + 인트로 무브
```

## 12. 성능 예산 (60fps 목표)

- 드로우콜 < 100: 나무·풀·집 등 반복물은 전부 `InstancedMesh` 또는 지오메트리 병합
- `renderer.setPixelRatio(min(devicePixelRatio, 1.5))`
- 그림자 맵 1장(2048), 플레이어 추적 shadow camera
- 파티클은 인스턴스 쿼드로 통합 관리

## 13. 구현 순서

1. 레포 클론 → 뼈대(index.html, main.js) → 빈 씬 + 하늘 + 조명
2. palette/gradientMap → 지형 + 안개 (여기서 1차 스크린샷: 하늘·언덕·안개 색이 지브리인지 판정)
3. 구름 → 나무·자연 → 마을 → 바다
4. 캐릭터 조립 → 비행 조작 → 카메라
5. 파티클·바람·연출 → 시작 오버레이
6. **튜닝 루프**: Chrome 도구로 띄워 스크린샷 ↔ 참고 이미지 비교하며 색·안개·조명 반복 조정 (가장 중요한 단계)
7. README(스크린샷 + 조작법) 작성, 커밋. push는 확인 후

## 14. 검증

- Chrome 브라우저 도구로 실행: 콘솔 에러 0건, 스크린샷으로 무드 판정
- 키 입력 시뮬레이션으로 비행 경로 확인: 이륙 → 마을 상공 → 시계탑 선회 → 대왕나무 → 바다·등대 → 구름 통과
- 5분 방치 후에도 프레임 유지(파티클/구름 누수 없음)
